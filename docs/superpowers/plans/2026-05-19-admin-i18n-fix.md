# Admin i18n Translation Fix — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [x]`) syntax for tracking.

**Goal:** Fix every hardcoded English/Arabic string in all admin pages so the Arabic locale shows only Arabic and the English locale shows only English.

**Architecture:** All translation keys live in `src/locales/en.ts` (source of truth with `MessageSchema` type) and `src/locales/ar.ts` (satisfies the schema). Views consume translations via `useI18n()`'s `t()` function or the template `$t()` global. The fix is: (1) add missing keys to both locale files, then (2) replace every hardcoded string in views with the corresponding `t()` call.

**Tech Stack:** Vue 3 Composition API, vue-i18n v9, TypeScript
